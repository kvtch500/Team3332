## What This Session Did

**1. Native sync to iOS (DONE):**
- `cd mobile && npm run sync` → copied `app/` → `www/` → `npx cap sync` → pod install (31s), all 5
  Capacitor plugins resolved, sync finished. `npm run open:ios` → Xcode. Built to **iPhone 17 Pro

**2. Fixed: Record screen crashed → black screen on iOS (native-only):**
- Symptom: app loaded fine, but tapping **Record** blanked the whole app. Safari Web Inspector
  (Develop → Simulator → TEAM 3332) showed the real error:
  `TypeError: Cap.registerPlugin is not a function … in <RecordRun>`.
- Root cause: the native bridge injects `window.Capacitor` (so `isNativePlatform()` works) but
  **NOT** `registerPlugin` — that normally comes from the `@capacitor/core` JS module, which this
  no-build single-HTML app never loads. Code took the "native" path, called `registerPlugin`, threw,
  and with no error boundary the whole tree blanked.
- Fix (`app/index.html`, GeoTracker IIFE ~line 1783): added `canUsePlugins` guard so the native
  plugin path only runs when `registerPlugin` actually exists; otherwise it cleanly falls back to
  `navigator.geolocation`. Info.plist already has the location usage strings, so **foreground** GPS
  works. Recorder now renders (verified in sim: "WALKING / 0.00 MILES / timer running / END").

**3. Fixed: blank map basemap on iOS (native-only):**
- Network tab showed `api.mapbox.com` tile requests in **red = failed**. The Mapbox token is
  URL-restricted to team3332.com; the native app runs from `capacitor://localhost`, an origin Mapbox
  rejects (403) → blank basemap. (URL-restricted tokens fundamentally don't work for native apps.)
- Fix (`app/index.html`, map config ~line 1879): added `IS_NATIVE_APP` check; `USING_MAPBOX` is now
  false inside the Capacitor app, so the native app uses **OpenStreetMap** tiles (no token, no
  restriction — already muted by the `.tiles-osm` CSS to fit the dark theme). Web (team3332.com) is
  unchanged and keeps the dark Mapbox style. Map now renders on device.
- Decision (Ernest): OSM fallback on native. Optional future upgrade = a separate non-restricted
  Mapbox mobile token if he wants the dark style back on mobile.

**4. Backend regression tests (DONE earlier this session):**
- Added `backend/test/auth.test.js` — 11 tests: PATCH /auth/me partial-body COALESCE preservation
  (name-only / empty / single / multi-field), avatar_url round-trip + 413 size/type guards, GET/PATCH
  auth guards. Full suite **104 passed / 0 failed** (was 93).
